# 更新与版本

## 查看当前版本

打开“设置 > 关于”查看当前版本、系统信息和项目链接。反馈问题时，请同时提供版本号、操作步骤和是否能够稳定复现。

## 安装更新

从项目发布页面下载与你的平台和架构匹配的安装包。安装前先退出正在运行的 SpringNote，尤其是使用桌面组件、托盘或正在同步时。

安装程序会更新应用文件，不会主动删除数据目录中的日报、周报、月报、图片和配置。数据目录是否存在其他备份由用户的系统和存储策略决定。

## 更新后检查

首次启动新版时，应用可能需要检查已有数据或准备新的功能。进入主界面后可以检查：

1. 数据目录仍然正确。
2. 最近的日报和图片可以打开。
3. 供应商和默认模型仍然存在。
4. 快捷键、托盘和桌面组件符合预期。

如果升级后某项 AI 功能不可用，先检查供应商连接和默认模型，不要直接删除原有配置。

## 更新日志

更新日志记录新增功能、行为调整、兼容性变化和问题修复。升级前阅读对应版本可以提前了解设置是否需要重新确认。
