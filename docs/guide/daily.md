# 日报

日报以自然日为单位保存当天的工作记录，是笔记本中最基础的记录类型，也是周报、回忆书和部分统计功能读取的来源。日报文件可以由首页输入流程创建，也可以在笔记本中直接编辑和保存。

## 内容来源

日报正文可能包含以下内容：

- 首页提交的文字记录；
- 首页提交后保存的数据目录图片及其 Markdown 引用；
- 首页智能生成得到的栏目整理内容；
- 笔记本中的手动编辑内容；
- AI 日报合并返回的 Markdown。

首页选择的普通文件目前只把文件名和路径加入输入，不会复制文件、读取内容或创建附件。图片和普通文件的处理规则不同：图片可以保存为笔记引用，普通文件仍只是输入中的路径信息。

如果 AI 日报合并成功，正文使用模型返回的 Markdown；没有可用模型、请求失败或返回内容不可用时，使用应用内的本地合并结果。应用不保证额外保存一份未经处理的原始输入副本。

## 创建时机

当天第一次提交首页输入时，应用会创建或更新当天日报。仅打开首页、打开笔记本、选择日期或查看列表不会创建空白日报。

已有日报再次提交内容时，应用会读取当前正文，并在现有内容基础上合并新增内容。笔记本中的手动编辑会直接保存到当前文件，并作为后续处理时的已有正文。

打开笔记本的日报类型时，如果当天文件不存在，页面会确保当天日报文件存在并选中它。其他日期的日报不会因为浏览列表而被自动创建。

## 文件名称

日报文件使用以下格式命名：

```text
YYYY-MM-DD.md
```

文件名中的日期使用本地日期。例如 `2026-07-15.md` 表示 2026 年 7 月 15 日的日报。文件保存在数据目录的 `notes/daily` 文件夹中，数据目录更换后使用新的日报目录。

## 编辑与保存

选中日报后，编辑器读取该文件的完整 Markdown 正文。文本变化会自动保存，保存过程中页面显示保存状态；保存成功后同步更新列表标题、预览、修改时间和搜索索引。

读取或保存失败不会把失败内容当作成功保存结果。编辑器仍保留当前界面中的文本，具体错误状态由页面显示。

## 日期边界

日报按本地日期归档。跨过午夜后，新提交内容归入新的日期文件；前一天的日报不会因为日期变化而自动改名或迁移。
