# 快速输入框

快速输入框用于保存当天的文字和图片。提交时，文字会写入当天日报，图片会复制到当前数据目录并插入 Markdown 图片链接；文件不会被复制到数据目录，当前只会把文件名和原始路径作为本次输入中的路径信息。

## 图片

图片可以来自文件选择器或系统剪贴板。添加后先显示在待提交区域，提交成功后才成为当天日报的一部分。图片预览依赖保存后的引用路径；原始图片文件不会因为复制到数据目录而被删除。

智能生成是否能够理解图片取决于所选模型是否支持图片输入。模型不支持图片时，图片仍然保存，只有 AI 输入中不包含图片内容。

## 文件

文件选择目前只生成当前输入中的文件路径信息。提交后，文件名和路径会作为文字内容的一部分参与日报整理和 AI 请求，但应用不会读取文件内容、复制文件或建立可供笔记预览的附件引用。

文件路径能否被模型访问取决于模型服务和运行环境；把路径发送给模型不代表模型已经读取文件，也不代表文件已经保存到 SpringNote 数据目录。文件内容解析和真正的附件管理目前尚未实现。

## 智能生成

智能生成会读取当前输入内容，并按照设置中的三个首页栏目定义返回结构化结果。每个结果包含对应栏目中的内容条目，首页根据栏目配置显示标题和内容。

结构化结果会保存为当天的首页概览，并参与当天日报的生成。日报最终内容取决于 AI 日报合并是否成功：合并成功时使用模型返回的 Markdown；合并失败或没有可用模型时，使用应用内的本地合并结果。应用不保证在 AI 合并成功时额外保留一份未经处理的原始输入副本。

再次生成时，新结果会与当天已有概览合并，已有栏目内容不会因为一次空结果而被无条件清空。没有模型、供应商不可用、请求失败或结构化结果不符合栏目定义时，文字和已保存的图片仍然保留。
