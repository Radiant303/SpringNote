# 周报

周报以一周为单位整理阶段成果和问题，与来源日报分开保存。生成或编辑周报不会反向修改这一周的日报。

## 内容来源

周报生成时读取目标周一至周日范围内有实际内容的日报，并将这些日报按日期组合后交给报告模型。没有有效正文的日报不会为周报提供内容。

周报生成后的 Markdown 会保存为独立文件，可以在笔记本中继续编辑。手动修改属于周报自身内容，不会写回来源日报；重新生成时，系统仍会按目标周读取来源日报。

## 创建时机

周报可以由 AI 报告生成流程创建，也可以在笔记本中首次保存时创建。生成前会检查目标周是否已有有实际内容的周报，已有周报不会被启动时的自动生成流程覆盖。

应用启动时会检查已经结束、且存在有效日报的历史周。只有来源内容存在、目标周报没有有效正文并且 AI 报告生成成功时，才会创建缺失周报。当前周不会在启动检查中作为已完成周处理。

生成失败、模型不可用、来源为空或返回空内容时，不会用空结果覆盖已有周报。

编辑器头部的“重新生成”按钮可按目标周重新读取来源日报、调用报告模型并覆盖当前周报文件，不受启动流程“已有内容不覆盖”的限制。生成期间按钮显示进度、编辑器暂时不可编辑；失败时错误信息显示在头部左侧的状态条中。该周没有有效日报内容时不执行生成。

## 文件名称

周报文件使用 ISO 周历命名：

```text
YYYY-Www.md
```

例如 `2026-W29.md` 表示 ISO 年份 2026 年第 29 周。周一是周的起始日，周日是结束日。跨年周按照 ISO 周所属年份命名，因此自然年年初或年末的周编号可能属于相邻的 ISO 年份。

文件保存在数据目录的 `notes/weekly` 文件夹中。

## 编辑与读取

笔记本列表按周报文件名倒序显示。选中后读取完整 Markdown 正文，编辑内容会自动保存，并更新标题、预览、修改时间和搜索索引。
