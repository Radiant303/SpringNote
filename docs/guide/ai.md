# 默认模型

默认模型页面为三个独立用途分别指定模型：智能生成、编辑补全和回忆书。每个用途只能有一个当前选择，但三个用途可以选择不同供应商和不同模型。

## 模型选择器

点击某一用途的选择区域后打开模型选择器。模型按供应商分组显示，选择器支持按模型显示名称、模型 ID 或供应商名称搜索。

选择器中的“未选择”是一个明确状态。选择后，应用保存供应商和模型的关联；仅保存模型名称而没有对应供应商时，不能形成可用的默认模型。

模型选择列表只显示当前已配置的模型。编辑补全模型列表还会额外筛选模型类型，只有标记为支持“补全”的模型才会出现。

## 智能生成模型

智能生成模型用于首页内容整理、日报合并，以及周报和月报的 AI 生成。选择可用模型后，相关功能才会向该模型发送请求。

### 已选择模型

应用会检查模型所属供应商是否启用、API Key 是否存在，以及模型是否仍在供应商的模型列表中。检查通过后：

- 首页可以执行智能生成；
- 日报可以使用 AI 合并；
- 周报和月报可以请求 AI 初稿；
- 支持图片输入的模型可以接收首页粘贴的图片。

### 未选择模型

首页文字和已保存图片仍可以进入本地日报处理流程，但不会执行智能生成请求。日报使用本地合并逻辑生成可保存内容；周报和月报不会生成 AI 初稿。

## 编辑补全模型

编辑补全模型用于笔记本编辑器中的 AI 实时补全。选择器只提供模型类型中标记为“补全”的模型，因为实时补全使用的是专用的补全请求方式。

### 已选择模型

编辑器会在满足补全触发条件时请求该模型。接受返回内容后，建议插入当前光标位置；忽略、取消、超时或请求失败不会修改原文，也不会影响撤销和保存。

要真正可用，模型还必须同时满足以下条件：

- 所属供应商已启用；
- 供应商 API Key 不为空；
- 供应商使用 OpenAI-compatible 协议；
- 模型仍存在，并保留“补全”类型标记。

### 未选择模型

编辑器不会发起 AI 补全请求，但普通输入、Markdown 高亮、撤销、重做、保存和预览均不受影响。

## 回忆书模型

回忆书模型用于根据关键词搜索结果、指定日期记录、周报和月报内容生成 AI 回答。它不负责本地搜索本身，搜索和记录读取在没有模型时也可以执行。

### 已选择模型

应用会先确认模型所属供应商已启用、API Key 不为空且模型仍存在。检查通过后，回忆书可以在检索结果基础上继续生成回答，并按照回忆书中选择的思考等级发送请求。

### 未选择模型

回忆书仍可以执行关键词搜索和读取指定记录，并显示匹配结果或 Markdown 内容；但不会生成完整的 AI 分析、归纳和追问回答。

## 模型失效

默认模型选择保存后，如果供应商被关闭、API Key 被清空、模型被删除或模型类型被取消，配置中的选择不会自动替换为其他模型，但对应功能会进入不可用状态并显示原因。

重新启用供应商、补充 API Key、恢复模型或重新选择可用模型后，功能可以恢复。更换某一用途的默认模型不会影响另外两种用途。
