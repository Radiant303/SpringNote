# 数据概览

数据概览中的数值来自当前数据目录内的记录和统计数据。它们是应用状态的汇总，不会反向修改日报、周报或月报。经验进度、时薪与收益以及活跃热力图分别使用不同的统计口径。

## 经验进度

经验进度由有效的首页整理次数累计产生。一次有效整理是指首页生成请求成功并被记录为当天的首页生成；普通打开首页、保存没有触发整理的内容、请求失败都不会增加经验。

每天最多计入 10 次首页整理。当天超过 10 次的成功整理仍可以更新首页内容，但不会继续增加经验，因此重复生成不会无限提高进度。

经验总值按每 100 点划分等级。进度显示当前等级内已经累计的部分，达到下一个 100 点后进入下一级，并重新从当前等级起点计算进度。统计读取失败时，页面使用可用的本地状态，不会阻止保存日报。

## 时薪与收益

收益来自桌面组件的工作计时。计时处于运行状态时，每经过一秒会根据当前设置累加对应收益；暂停后不再增加工作时长和收益。

收益计算规则为：

```text
每秒收益 = 每日薪资 ÷ (每日工作时长 × 3600)
当前收益 = 每秒收益 × 当天累计计时秒数
```

每日工作时长小于或等于 0 时，计算使用 8 小时作为默认工作时长。收益按本地日期分别累计，跨日后当天计时和当天收益重新开始；历史累计收益仍保存在统计中。

首页显示当天收益和累计总收益。实时计时中的未写入部分也会暂时显示，统计写入失败不会让计时停止；应用退出时会尝试保存尚未写入的计时数据。

## 活跃热力图

活跃热力图显示最近 140 天的活跃情况，每个日期对应一个活跃次数。活跃次数来自当天的首页整理和成功的编辑补全等统计事件，不是简单按照 Markdown 文件是否存在判断。

热力图按本地日期统计。跨午夜后，新活动归入新的日期；历史记录删除不会自动制造新的活跃事件，但刷新统计后相关文件数量和活动数据可能发生变化。

| 当天活跃次数 | 显示等级 |
| ---: | --- |
| 0 | 无活动 |
| 1–2 | 低 |
| 3–4 | 中 |
| 5–7 | 高 |
| 8 次及以上 | 最高 |
