# 存储管理

存储管理用于处理已经不再被笔记引用的图片。它采用“扫描、确认、清理”的流程，避免因为删除了 Markdown 图片链接就误删文件。

## 图片扫描

点击图片扫描后，应用会检查数据目录中的图片，并查找仍被日报、周报或月报引用的文件。扫描阶段只读取和整理结果，不会修改或删除文件。

扫描读取开始时的数据状态；扫描期间发生的笔记保存可能不在本次结果中，需要再次扫描才能得到新的引用状态。

## 图片预览

预览列表用于确认候选图片到底是什么内容。选择图片后，可以查看缩略图、文件信息和引用状态；看不清用途的图片不要直接删除，应先回到相关笔记确认。

## 清理未使用图片

1. 扫描完成后查看候选列表。
2. 预览图片并确认不再需要。
3. 选择要清理的图片。
4. 点击清理并确认删除。

清理只针对应用数据目录内被判定为未使用的图片，不会扫描或删除用户其他目录文件。删除后无法依靠 SpringNote 恢复，因此重要图片应先备份。
