# 供应商

供应商页面用于管理 AI 服务连接和模型来源。页面左侧列出已保存的供应商，右侧显示当前供应商的连接信息、启用状态和模型列表。供应商搜索只匹配供应商名称，不搜索 API 地址、模型名称或密钥。

## 供应商列表

供应商列表显示名称、启用状态和当前模型数量。点击列表项后，右侧切换到对应供应商的详细配置；选中状态只改变当前查看对象，不会修改供应商配置。

搜索框会根据供应商名称进行筛选，查询不区分英文字母大小写。没有匹配项时显示空结果；清空搜索后恢复完整供应商列表。搜索不会触发网络请求，也不会修改供应商数据。

## 添加供应商

添加供应商时可以先选择内置模板。模板会预填供应商名称、协议对应的默认地址、API 路径和部分示例模型，保存前仍可以修改这些内容。当前模板包括 OpenAI、OpenAI Responses、DeepSeek、Qwen DashScope、Kimi、OpenRouter、SiliconFlow、Ollama、Google 和 Claude。

添加表单包含：

- **是否启用**：决定添加后能否参与 AI 请求；
- **名称**：本地显示名称，也是供应商搜索使用的名称；
- **API Key**：服务认证信息，以隐藏形式显示；
- **Base URL**：服务的基础地址；
- **API 路径**：具体请求路径，Gemini 等协议可能不使用该字段。

添加供应商只保存本地配置，不会自动测试连接，也不会自动把它设为首页、补全或回忆书的默认模型。

## 连接配置

供应商详情可以修改名称、API Key、协议、API Base URL 和 API 路径。协议决定请求格式和部分默认地址：OpenAI-compatible 用于兼容聊天或 Responses 请求，Gemini 和 Claude 使用各自的请求格式。

修改连接字段会保存到应用配置。保存成功不等于服务一定可用，实际可用性还取决于地址、认证、模型 ID、协议和服务端权限。

## 启用与关闭

供应商启用开关控制该供应商是否参与新的 AI 请求。关闭供应商不会删除连接信息、模型列表或 API Key；重新启用后，满足其他条件的默认模型可以继续使用。

如果某个默认模型属于已关闭的供应商，智能生成、编辑补全或回忆书对应功能会显示模型不可用，不会自动切换到另一个供应商。

## 测试连接

测试连接会使用当前供应商和模型配置发起实际连接测试。供应商至少需要有一个模型；没有模型时，页面会先提示添加模型。

测试结果反映当前服务地址、认证信息、协议和选定模型能否完成请求。测试成功不代表所有模型都支持图片输入、工具、推理或编辑补全；这些能力由模型自身的能力标记和服务端实际支持共同决定。

测试失败不会删除配置、模型或笔记。常见失败原因包括地址错误、API Key 无效、供应商未授权模型、网络失败、协议不匹配和服务端返回错误。

## 获取模型

获取模型会请求供应商提供的模型列表，并在弹窗中展示可添加或移除的模型。模型列表可以按模型名称、模型 ID 或供应商名称进行筛选，并按模型 ID 的前缀分组显示。

获取到的模型不会全部自动成为本地模型。用户在列表中添加后，模型才会写入供应商配置；已经添加的模型可以从获取结果中移除。获取失败时，已有本地模型不会被清空。

部分供应商不提供模型列表，或者接口返回格式不符合服务支持范围，此时获取模型会失败，但仍可以通过添加模型手动登记。

## 添加模型

手动添加模型需要填写模型 ID，显示名称可以留空；留空时使用模型 ID 作为显示名称。添加模型只建立本地选择项，不会验证该模型一定存在或可调用。

已添加模型可以编辑以下信息：

- 显示名称；
- 模型类型：聊天、补全；
- 输入模式：文本、图片；
- 能力：工具、推理。

这些标记会影响默认模型选择器和功能可用性。例如，编辑补全选择器只显示标记为“补全”的模型；图片输入和思考模式也会检查对应能力。标记与服务端真实能力不一致时，请求仍可能失败。

## 模型删除

删除模型会从当前供应商的本地模型列表移除该模型。已经把该模型选为默认模型的功能会变为未选择或不可用状态，不会自动改用其他模型。

删除模型不会删除服务端模型，也不会修改已经生成的日报、周报、月报和回忆书历史消息。

## 删除供应商

删除供应商前会显示确认弹窗。确认后会移除该供应商的连接配置、启用状态和关联模型；依赖这些模型的默认模型选择会失效。

删除供应商只影响 SpringNote 保存的本地配置，不会删除服务端账户、远端模型、已有笔记或图片。删除操作不可通过供应商页面撤销，需要重新添加并配置供应商。
