# AI 能力介绍

回忆书用于从已经保存的日报、周报和月报中查找信息，并根据查找到的内容生成回答。它由本地检索、记录读取和 AI 回答三个环节组成，每个环节承担不同职责。

## 关键词检索

回忆书可以按关键词搜索全部记录，也可以限定只搜索日报、周报或月报。关键词搜索返回匹配的文档、文档类型、名称和预览信息，用于定位可能相关的记录。

关键词搜索至少需要两个字符，多个关键词按任一词命中处理。搜索结果受到回忆书检索设置中的数量和上下文长度限制；搜索结果本身不是完整文档，命中的记录需要进一步读取才能获得完整 Markdown 内容。

## 按时间读取

当问题包含明确日期或周期时，回忆书可以直接读取对应记录：

- 按自然日期读取某一天的日报；
- 按 ISO 年份和周数读取周报；
- 按自然年份和月份读取月报；
- 读取与指定月份日期范围重叠的周报集合。

读取操作返回目标记录的完整 Markdown 内容。月内周报按照日期范围判断，一周只要有日期落在目标月份内，就可能被纳入该月份的结果，因此跨月周可能同时出现在相邻月份的查询中。

## AI 回答

回忆书通常先判断问题是否包含明确的日期、周或月份线索，再决定直接读取记录或执行关键词搜索。获得相关文档后，系统把检索结果或完整记录作为回答上下文交给所选模型，由模型进行归纳、比较、提取和说明。

AI 回答只使用本地检索或读取到的内容作为记录依据。没有检索到相关历史 Markdown 时，回答无法获得对应的本地事实；模型可以进行语言组织，但不会因为本地没有记录而创建一份新的历史记录。

## 数据边界

回忆书的搜索和读取不会修改日报、周报或月报。AI 生成的回答、思考内容和工具过程属于回忆书会话，不会自动写回笔记文件。只有在笔记本中编辑并保存时，Markdown 正文才会发生变化。

没有配置回忆书模型时，本地关键词搜索和指定记录读取仍可执行，可以查看匹配记录和完整 Markdown 内容；但不会生成 AI 分析、总结或追问回答。供应商连接失败、模型不支持当前请求或请求被取消时，只影响当前回答，已有搜索结果、本地文档和历史对话不会被修改。
