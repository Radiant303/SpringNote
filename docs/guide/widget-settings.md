# 组件设置

组件设置用于控制桌面组件是否显示、以什么形态显示，以及组件计时和位置的基本行为。桌面组件是主应用之外的独立窗口，当前支持 Windows 和 macOS；不支持的平台会显示平台限制。

## 显示桌面组件

“显示桌面组件”控制组件窗口是否存在于桌面上：

- 开启时，组件窗口可以显示并接收拖动、计时控制等操作；
- 关闭时，组件窗口隐藏，不再显示在桌面上；
- 隐藏窗口不会自动清除当天计时，也不等同于暂停或停止计时。

组件是否显示与主应用窗口是否打开相互独立。主应用最小化、关闭或进入托盘时，组件可以继续按照自身状态运行。

## 圆球模式

“桌面组件圆球模式”控制组件的默认显示形态：

- 开启时，组件以圆球状态显示，占用较小的桌面空间；
- 关闭时，组件以展开状态显示计时和收益等信息；
- 圆球和展开状态之间可以在组件窗口中切换，切换不会重置计时。

圆球模式只有在组件已启用且当前平台支持桌面组件时才可操作。切换显示形态不会修改笔记、统计记录或个人信息。

## 鼠标操作

组件窗口的鼠标操作在 Windows 和 macOS 上保持一致：

- **左键单击**：切换工作计时状态。运行中的组件会暂停，暂停或停止状态会开始计时；单击圆球不会打开主应用。
- **右键单击**：打开 SpringNote 主窗口，并切换到首页。
- **左键拖动**：移动组件窗口。按下后移动超过很小的位移阈值才会判定为拖动；拖动结束后保存位置，不会同时触发左键计时切换。
- **圆球悬停**：指针进入圆球有效区域时展开组件，指针离开有效区域后收起为圆球。展开状态下移动指针不会因为经过矩形窗口边界而立即收起。

鼠标按下后如果没有发生实际位移，才会作为单击处理；发生拖动、鼠标捕获被系统取消或操作被中断时，不会误切换计时状态。圆球的透明区域不作为有效点击区域，点击必须落在圆球形状内部。

右键操作只负责打开主窗口，不改变计时状态。主窗口打开后，组件仍保持原来的运行、暂停或停止状态。

## 工作计时

组件显示当前计时状态、当天累计工作时长和当天收益。计时开始后，经过的时间会逐步累加；暂停或停止后不再累加。

隐藏组件、关闭主窗口或切换圆球模式不会自动暂停计时。计时状态由组件控制器维护，组件重新显示后会继续展示当前状态。

计时数据会写入统计记录，并同步到首页的时薪与收益区域。正在运行但尚未提交到统计记录的短暂计时会先显示在当前界面，应用退出时会尝试保存这部分计时。

## 收益规则

收益根据偏好设置中的每日工作时长和日薪换算：

```text
每秒收益 = 每日薪资 ÷ (每日工作时长 × 3600)
当天收益 = 每秒收益 × 当天累计工作秒数
```

每日工作时长小于或等于 0 时按 8 小时计算。收益按本地日期累计，跨过午夜后当天计时和收益重新开始，历史累计统计继续保留。

修改每日工作时长或日薪只影响之后的计时换算，不会重算已经保存的历史收益，也不会修改日报内容。

## 拖动与位置

组件窗口可以直接拖动。拖动完成后位置会保存到配置，重新启动应用后恢复上一次位置。位置保存采用延迟写入，避免每次鼠标移动都写入配置文件。

多显示器环境下，组件位置会限制在可见工作区内。显示器数量、分辨率或缩放比例变化后，如果原位置不再可见，组件会被调整到可见区域，避免窗口出现在无法操作的屏幕外。

圆球和展开状态使用同一组件位置基准，切换状态不会把组件移动到另一个独立位置。

## 平台与异常状态

组件功能只在 Windows 和 macOS 上提供。平台不支持时，组件设置仍可显示配置状态，但相关控件不可用。

组件窗口暂时无法响应时，不会影响主应用中的笔记编辑、AI 请求和本地数据保存。窗口重新显示或重新连接后，会同步当前计时和显示状态。位置、显示开关或圆球模式与页面暂时不一致时，重新打开组件或重启应用会重新读取配置。
