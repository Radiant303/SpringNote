# AI 实时补全

AI 实时补全用于根据编辑器光标前后的 Markdown 内容预测后续文字。预测结果以编辑器中的临时补全显示，接受后才会写入正文；未接受的预测不会改变文件内容。

## 触发条件

补全只在当前有选中文档、编辑器已经加载完成、光标为单一插入位置且没有选中文本时触发。用户停止输入或光标变化后，应用等待约 300 毫秒再发起请求；等待期间继续输入会取消旧请求并重新计时。

请求发送前会检查编辑补全模型是否可用。未选择模型、供应商未启用、API Key 为空、模型不存在、模型没有“补全”类型标记，或供应商协议不支持补全时，不会发起有效补全请求。

## 上下文

请求使用光标前的内容作为前缀，光标后的内容作为后缀，让模型在当前文档结构中生成中间缺失部分。补全面向 Markdown 编辑内容，不会自动把其他日报、周报或月报作为上下文加入请求。

## 接受方式

有预测内容时：

- `Tab` 接受全部预测；
- `Ctrl+L` 接受预测的当前行；
- `Ctrl+K` 接受预测的当前字符。

Windows 使用上述 Ctrl 组合；macOS 使用对应的系统修饰键。没有预测内容时，`Tab` 保持编辑器本身的制表符输入行为。

## 状态与保存

请求过程中显示预测状态，返回内容为空时显示没有可用预测。请求失败、取消、超时或用户继续编辑时，临时预测会被清除，原文保持不变。

接受预测后，插入内容会按照普通编辑处理，进入撤销历史并自动保存。撤销、重做、切换文档或切换工作区模式不会把未接受的预测写入其他文档。
